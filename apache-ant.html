Apache ANT

This page contains QnA on Apache Ant from TutorialsPoint.com

Ant is a java based build tool developed by Apache for the purpose of automating the build and deployment process.

Another Neat Tool

Following are the activities that are done by a developer manually after writing the application code:-
  1. Compiling the application code
  2. Packaging the binaries (class files)
  3. Deploying the packaged binaries to test server. (Note that this is not the PROD server)
  4. Testing the application code on the test server
  5. Copying the application code from test server to the prod server.
All of the above activities were done by the developer manually onne after the other. With automated build and deployment tool like Apache Ant, there is no further need to do all these steps manually.
Follow the steps below:-

1. Goto website https://ant.apache.org/

2. Click on the link to download Ant in binary form "https://ant.apache.org/bindownload.cgi."



3. Click on tar.gz link (since you are using macbook).



4. Tarred (zipped) file gets downloaded onto your macbook.



5. Double click on the tarred file to extract it. You get a ant folder.



6. Open terminal and run the command:- export ANT_HOME=/Users/ngksh/Downloads/apache-ant-1.9.14 where "/Users/ngksh/Downloads/apache-ant-1.9.14" denotes the path to the extracted folder in step 5.

7. Run command:- export PATH=${PATH}:${ANT_HOME}/bin. Note that "bin" folder is present within the extracted folder.

8. Run command:- ant -version. You should see the version of ant installed on your macbook.

2. Ant's build file should be present in the base directory of the project.
No, the above 2 are the best practises and it is NOT mandatory to follow these conventions.



1. Save Ant's build file (with xml extension). As per the industry convention, the name of Ant's build file should be "build.xml".

2. Open terminal.

3. Navigate to the path where you have saved the Ant's build file.

4. Type the following command to run the Ant's build file:-
ant

When you type the command ant on terminal, the system starts searching for the file with name "build.xml" in the path where the command was executed. When the file is found, the instructions written within the file are executed.

name:- This attribute is marked as optional by Ant for project tag. Its value can be any user defined string.

default:- This attribute is marked as mandatory by Ant for project tag. Its value should be same as the name of any one target tag present in the build file.

basedir:- This attribute is marked as optional by Ant for project tag. Its value should be the path of the projects base directory.


name:- This attribute is marked as mandatory by Ant for target tag. Its value can be any user defined string.

description:- This attribute is marked as optional by Ant for target tag. Its value can be any user defined string.

depends:- This attribute is marked as optional by Ant for target tag. Its value should be a comma-separated list of names of targets on which this target depends.

if:- This attribute is marked as optional by Ant for target tag. Its value should be name of a property (user created property/inbuilt Ant property). The tasks written within the target get executed only if the value of the property is set

unless:- This attribute is marked as optional by Ant for target tag. Its value should be name of a property (user created property/inbuilt Ant property). The tasks written within the target get executed only if the value of the property is not set
